🟢 Health
GET/healthSystem status — no auth required
cURL
curl http://localhost:8000/health
Response 200
{ "status": "online", "version": "2.1.0", "database": "connected", "ml_model": "loaded" }
🔐 Authentication
POST/auth/tokenGet JWT access token
FieldTypeNote
usernamestringRequired
passwordstringRequired
cURL
curl -X POST http://localhost:8000/auth/token \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
Response 200
{ "access_token": "eyJ...", "token_type": "bearer" }
🚌 Fleet Tracking
POST/locationGPS ping from bus — X-API-Key required
FieldTypeNote
vehicle_idstringRequired
route_idstringRequired
latfloatRequired — -90 to 90
lngfloatRequired — -180 to 180
speedfloatRequired — km/h ≥ 0
passenger_countintegerOptional
cURL
curl -X POST http://localhost:8000/location \
  -H "X-API-Key: sim-key-change-me" \
  -H "Content-Type: application/json" \
  -d '{"vehicle_id":"BUS-01","route_id":"RT-101","lat":31.52,"lng":74.35,"speed":42}'
POST/location/telemetryIoT passenger count — X-API-Key required
FieldTypeNote
vehicle_idstringRequired
passenger_countintegerRequired — 0 to 200
cURL
curl -X POST http://localhost:8000/location/telemetry \
  -H "X-API-Key: sim-key-change-me" \
  -H "Content-Type: application/json" \
  -d '{"vehicle_id":"BUS-01","passenger_count":28}'
GET/buses/liveAll active buses with live positions
cURL
curl http://localhost:8000/buses/live
Response 200
[{ "vehicle_id":"BUS-01", "lat":31.52, "lng":74.35, "speed":38, "passenger_count":24 }]

🌐 GTFS Pipeline
POST/admin/gtfs/demoIngest demo GTFS feed — Bearer token required

⚡ WebSocket
WSws://localhost:8000/ws/busesReal-time fleet stream
JavaScript
const ws = new WebSocket("ws://localhost:8000/ws/buses");
ws.onmessage = (e) => {
  const msg = JSON.parse(e.data);
  // msg.type === "bus_update" → GPS position
  // msg.type === "telemetry"  → passenger count
};
